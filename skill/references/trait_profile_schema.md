# Trait Profile Schema

`state/trait_profile.md` is the living knowledge document that teaches
every downstream extractor how this trait is reported in the literature.
It is written by the `trait_learner` subagent and updated periodically.

## File format

Markdown with a YAML frontmatter and exactly ten sections in a fixed
order. The format is stable; downstream scripts parse it by section
header.

```markdown
---
trait: <canonical name>
taxa: <taxonomic scope>
version: <iso timestamp>
seed_papers: [<sha256>, ...]
batches_observed: <int>
---

# Trait Profile

Auto-generated by trait_learner. Human edits above the "--- AUTO ---"
line below are preserved across updates.

--- AUTO ---

## 1. Canonical Name and Synonyms
- Canonical: <string>
- Synonyms: [<string>, ...]
- Abbreviations: [<string>, ...]
- Symbols: [<string>, ...]

## 2. Notation Conventions
Per observed notation:
- Notation: `<exact pattern>`
- Example verbatim: `<quote>`
- Regex: `<optional regex>`
- Seen in: [<sha256>, ...]

## 3. Units and Variants
Per unit:
- Unit: <name>
- Variants: [<string>, ...]
- Conversion to canonical: <formula or note>

## 4. Valid Biological Ranges
Per taxon or global:
- Scope: <taxon or "global">
- Minimum: <value>
- Maximum: <value>
- Typical: <value>
- Citation evidence: [<sha256>:<page>, ...]

## 5. Common Confusions
Per confusion pair:
- Label A: <name>
- Label B: <name>
- How authors conflate them: <sentence>
- Disambiguating cue: <sentence>
- Worked example: <quote>

## 6. Reporting Structures
- Location: <text|table|abstract|figure|supplementary>
- Reliability: <high|medium|low>
- Notes: <sentence>

## 7. Qualifiers
- Qualifier: <sex|stage|tissue|population|other>
- How reported: <sentence>
- Default when unspecified: <value or "unknown">

## 8. Compilation vs Primary Signals
- Signal: <sentence>
- Example compilation verbatim: <quote>
- Example primary verbatim: <quote>

## 9. Edge Cases
- Case: <name>
- Description: <sentence>
- Author flag: <how they mark it>
- Worked example: <quote>

## 10. Per-Column Extraction Hints
Per schema column:
- Column: <name>
- Rule: <sentence the extractor obeys>
```

## Design constraints

- **Bounded size.** Keep total under 4,000 tokens so it fits in every
  subagent's context without bloating the prompt. If sections grow
  too long, the trait_learner should consolidate examples.
- **Human-editable top.** The Manager may add clarifications via
  `AskUserQuestion` above the `--- AUTO ---` line. The trait_learner
  must preserve that block verbatim across updates.
- **Evidence-linked.** Every claim in the profile must point to at
  least one `sha256:page` pair in the seed set. No unsupported
  generalizations.
- **Versioned.** Every update bumps the YAML `version` field and adds
  to `batches_observed`. The ledger stores `trait_profile_hash` per
  row, so later analyses can group results by profile version.

## Update semantics

In `mode: update`, the trait_learner:

1. Parses the current file.
2. Computes a diff against new observations from recent batches.
3. Adds only new notations, new synonyms, new edge cases. Does not
   rewrite existing sections unless contradictory evidence emerges.
4. On contradictions, preserves both observations and adds a
   `## 5. Common Confusions` entry describing the contradiction.
