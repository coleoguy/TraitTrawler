🧠 Manager Opus

Top-level orchestrator. Reads all state, makes strategic decisions, spawns worker agents. Handles user interaction, session lifecycle, and knowledge evolution review.

Opus thinks; Sonnet works. Manager context never accumulates PDFs or extraction reasoning.

🔍 Searcher Sonnet

Queries PubMed, OpenAlex, bioRxiv, Crossref. Triages papers into likely / uncertain / unlikely. Deduplicates against processed.json.

Writes → state/queue.json

Citation chaining & adaptive source ordering after 20+ queries.

📥 Fetcher Sonnet

Acquires PDFs via OA cascade: Unpaywall → OpenAlex → EuropePMC → Semantic Scholar → CORE → Proxy. Detects document type (table-heavy, prose, catalogue, scanned).

Writes → ready_for_extraction/
Fallback → leads.csv

📋 queue.json

Papers awaiting PDF fetch. DOI, title, abstract, triage score.

📋 ready_for_extraction/

Handoff JSON: PDF path, metadata, document type.

📋 leads.csv

Papers where PDF is unavailable.

🎯 Dealer Sonnet

Coordinates per-paper extraction. Picks handoff from queue, spawns Extractor (consensus or fast mode), handles Opus escalation on consensus failure.

⚡ Up to 2 concurrent Dealers
Moves handoff → state/dealt/

⚖️ Opus Escalation

When all 3 agents disagree on key numeric fields, Dealer spawns single Opus agent to adjudicate. If conf < 0.7 → needs_attention.csv.

🔬 Extractor Sonnet

Spawns 3 sub-agents in parallel, aligns records by primary key, applies majority-rule voting on all fields.

Consensus Voting

Strings: 2+ agree → accept; all differ → Agent A + flag
Numerics: 3 agree → full (+0.1); 2 agree → majority; all differ → flag
Confidence: median of 3 scores ± consensus bonuses/penalties

Writes → finds/{doi}_{ts}.json

🅰️ Agent A SONNET

Standard. Systematic extraction: tables first, then results & appendices. Two-pass for table-heavy papers.

🅱️ Agent B SONNET

Enumeration-First. Lists every species mentioned anywhere before extracting. Catches records missed by standard approach.

🅲 Agent C SONNET

Skeptical. Challenges every value — could it belong to a different species? Is notation ambiguous? Includes doubt_note field.

📋 finds/

Extraction results awaiting CSV write.

✍️ Writer Sonnet

Single-threaded only. Resolves taxonomy (GBIF), applies confidence calibration (isotonic regression), validates against rules, deduplicates, appends atomically via SchemaEnforcedWriter.

🔒 NEVER concurrent — single Writer only
Writes → results.csv (append-only)
Flags → needs_attention.csv

📊 results.csv

Final trait database. Append-only. Full provenance: source page, confidence, consensus type, agent disagreements, DOI, taxonomy resolution.
